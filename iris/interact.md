## 3. 数据结构

### 3.1 请求数据结构

#### 病理切片分析请求
- **Content-Type**: multipart/form-data
- **表单字段**:
  | 字段名 | 类型 | 必需 | 描述 |
  |--------|------|------|------|
  | file | File | 是 | 病理切片图片文件 |

#### 支持的文件类型
- .svs (Whole Slide Image)
- .tif, .tiff (TIFF 格式)
- .png, .jpg, .jpeg (常见图片格式)

### 3.2 响应数据结构

#### CaseData 对象
```typescript
interface CaseData {
  id: string;           // 病例 ID
  filename: string;     // 文件名
  malignancyScore: number; // 恶性度评分 (0-1)
  confidenceScore: number; // 置信度评分
  findings: AnalysisPoint[]; // 分析发现
  evidence: EvidenceItem[]; // 证据项
}

interface AnalysisPoint {
  category: 'Structure' | 'Cellular' | 'Invasion'; // 类别
  description: string;    // 描述
  citations: number[];    // 引用 ID
}

interface EvidenceItem {
  similarity: string;     // 相似度
  referenceId: string;    // 参考 ID
  source: string;         // 来源
  relatedFeatures: string; // 相关特征
}
```

#### 错误响应
```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述"
  }
}
```

## 6. 错误处理机制

### 6.1 前端错误处理

#### 网络错误
- **处理方式**: 捕获错误，返回 Mock 数据
- **用户体验**: 显示加载状态，然后展示 Mock 数据
- **日志记录**: 控制台输出警告信息

#### 业务错误
- **处理方式**: 显示错误信息，允许用户重试
- **用户体验**: 错误提示 + 重试按钮
- **日志记录**: 控制台输出错误详情

### 6.2 后端错误处理

#### 常见错误
| 错误类型 | 错误码 | 描述 | 处理方式 |
|----------|--------|------|----------|
| 文件格式错误 | 400 | 不支持的文件格式 | 返回错误信息 |
| 文件大小超限 | 413 | 文件大小超过限制 | 返回错误信息 |
| 分析失败 | 500 | 分析过程出错 | 返回错误信息 |
| 服务不可用 | 503 | 服务暂时不可用 | 返回错误信息 |

## 7. Fallback 机制

### 7.1 后端无响应时的处理

1. **前端检测**: Axios 捕获网络错误
2. **模拟延迟**: 模拟 1 秒网络延迟，提升用户体验
3. **返回 Mock 数据**: 使用 `MOCK_CASE` 作为 fallback
4. **状态更新**: 更新全局状态为 Mock 数据
5. **用户体验**: 用户看到的是 Mock 数据，无感知切换

### 7.2 Mock 数据结构

Mock 数据严格按照 `CaseData` 接口定义，包含：
- 基本信息 (id, filename)
- 评分 (malignancyScore, confidenceScore)
- 分析发现 (findings)
- 证据项 (evidence)
